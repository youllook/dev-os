# 07-notify.md — 通知與待機規則

## 觸發條件
- 情境06 DEPLOY 完成

## 執行角色
Planner（高階AI）

---

## 執行步驟

### Step 1 — 產生測試通知內容
自動彙整以下資訊產生通知：
```
【測試請求】
版本：{版本號} — {yyyyMMdd}
異動摘要：
  - #{IssueNo} {issue 標題}
  - #{IssueNo} {issue 標題}
  ...
測試重點：
  - {從 issue tickets 摘錄的關鍵行為}
  ...
請完成測試後回報結果。
```

### Step 2 — 通知人工測試
依 issue 勾選的通知方式發送（Slack / Email），進入待機狀態。

### Step 3 — 待機，等待測試回報
收到人工測試回報後，進行以下判斷：

---

**測試不通過**
- 請使用者描述問題
- 回報：
  ```
  收到，請回到情境02 ISSUE IMPORT
  依以下問題建立修復 issue：
    - {問題描述}
  ```

**測試通過 + 有新任務**
- 回報：
  ```
  收到，請提供新需求清單
  將進入情境02 ISSUE IMPORT
  ```

**測試通過 + 無新任務**
- 回報：
  ```
  本次迭代完成。
  版本 {版本號} 已歸檔於 release/{yyyyMMdd}-{版本號}
  系統待機，有新需求時請告知。
  ```

---

## 禁止事項
- 不得在未收到人工測試回報前自行判斷通過
- 不得自行建立修復 issue，交由情境02處理
